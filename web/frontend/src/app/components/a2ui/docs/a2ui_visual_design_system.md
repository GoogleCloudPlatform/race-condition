# A2UI Visual Design System (Glassmorphism)

## Premium Standards

A2UI components should feel like a native part of a modern, premium workspace.
The standard aesthetic is **Glassmorphism**, which emphasizes depth,
transparency, and clarity.

## CSS Tokens & Rules

### 1. The Glass Surface (Card/Group)

The base for any A2UI surface or grouped content.

- **Background**: `rgba(255, 255, 255, 0.03)`
- **Blur**: `backdrop-filter: blur(20px)`
- **Border**: `1px solid rgba(255, 255, 255, 0.08)`
- **Shadow**: `0 10px 25px -5px rgba(0, 0, 0, 0.3)`
- **Radius**: `16px` (Outer), `12px` (Internal)

### 2. Typography

- **Primary Text**: `#f1f5f9` (Off-white for readability)
- **Secondary Text**: `#94a3b8` (Muted gray)
- **Line Height**: `1.6` for optimal legibility.

### 3. Interactive Elements (Buttons)

Buttons should have a distinct, vibrant feel.

- **Gradient**: `linear-gradient(135deg, #3b82f6 0%, #2563eb 100%)`
- **Hover State**: `transform: translateY(-1px)` + stronger shadow.
- **Transition**: `all 0.2s cubic-bezier(0.4, 0, 0.2, 1)`

## Primitive Styling Mapping

| A2UI Primitive | Visual Realization                                               |
| :------------- | :--------------------------------------------------------------- |
| **Card**       | Glassmorphic container with shadow and padding.                  |
| **Column**     | Flex container with `flex-direction: column` and `gap: 0.75rem`. |
| **Row**        | Flex container with `flex-direction: row` and `gap: 0.75rem`.    |
| **Section**    | Vertical block with a subtle `border-bottom` divider.            |
| **Text**       | Clean, sans-serif typography with semantic weight support.       |

## Component Layout Patterns

### 1. Dashboard Logic

Use `Card` -> `Column` -> `Row` (Distribution: `spaceBetween`) to create a
header style with title and metadata. This pattern is ideal for summarizing
complex agent results (e.g., Scenario Completion Cards).

### 2. Interaction Pills (⚡/✓)

Pills provide visual feedback for sub-agent orchestration:

- **Call Pill (⚡)**: `background: rgba(59, 130, 246, 0.1)`,
  `border: 1px solid rgba(59, 130, 246, 0.2)`, `color: #60a5fa`.
- **Response Pill (✓)**: `background: rgba(34, 197, 94, 0.1)`,
  `border: 1px solid rgba(34, 197, 94, 0.2)`, `color: #4ade80`.

### 3. Progressive Consistency

Always provide a stable `messageId` for every part of a streamed response. This
allows the UI to reconcile partial updates into a cohesive layout without
flickering or element duplication.

## Generative Layout

Styling should be applied at the **primitive level** to ensure that any valid
layout generated by the model remains visually coherent and premium.
