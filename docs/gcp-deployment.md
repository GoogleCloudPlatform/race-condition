# Cloud Deployment Guide

> **Deploying from scratch?** See the
> [Fresh Deployment Guide](deployment/README.md) for a step-by-step walkthrough
> targeting a single GCP project, intended for external deployers.

Comprehensive guide for deploying the simulation backend to Google Cloud
Platform across multiple environments using Cloud Run and Vertex AI Agent
Engine.

## Prerequisites

- [Google Cloud SDK](https://cloud.google.com/sdk/docs/install) (`gcloud`)
- [Terraform](https://developer.hashicorp.com/terraform/install) (for
  infrastructure provisioning)
- Authenticated account (`gcloud auth login`)
- `gcert` (Googlers — required before builds or dependency installs)

## Architecture

### Deployment Targets

| Service        | Type             | Port | Target       |
| :------------- | :--------------- | :--- | :----------- |
| **gateway**    | Go (Gin)         | 8101 | Cloud Run    |
| **admin**      | Go (Gin)         | 8000 | Cloud Run    |
| **tester**     | Go BFF + Vite    | 8304 | Cloud Run    |
| **dash**       | Python           | 8301 | Cloud Run    |
| **simulation** | Python (FastAPI) | 8202 | Agent Engine |
| **planner**    | Python (FastAPI) | 8204 | Agent Engine |

### Domain Pattern

Each Cloud Run service gets a per-service custom domain:

```
{service}.{env}.keynote2026.cloud-demos.goog
```

Examples for dev:

- `gateway.dev.keynote2026.cloud-demos.goog`
- `admin.dev.keynote2026.cloud-demos.goog`
- `tester.dev.keynote2026.cloud-demos.goog`

Agent Engine services use Vertex AI managed URLs (not customizable).

## Service Discovery

Services discover each other using environment variables injected during
deployment. All URLs follow the per-service domain pattern:

| Variable         | Example Value                                        |
| :--------------- | :--------------------------------------------------- |
| `GATEWAY_URL`    | `https://gateway.dev.keynote2026.cloud-demos.goog`   |
| `ADMIN_URL`      | `https://admin.dev.keynote2026.cloud-demos.goog`     |
| `SIMULATION_URL` | _(Agent Engine managed URL, set post-deploy)_        |
| `PLANNER_URL`    | _(Agent Engine managed URL, set post-deploy)_        |

## Infrastructure Requirements

### Redis (Cloud Memorystore)

- **Purpose**: Session coordination, Switchboard message relay, DashLogPlugin
  event publishing
- **Tier**: Basic (dev), Standard HA (prod)
- **Network**: Must be accessible via VPC Connector

### Pub/Sub

- **Topics**: `agent-telemetry` (telemetry), `specialist-orchestration` (agent
  orchestration)
- **Subscriptions**: One per consumer service

### AlloyDB

- **Purpose**: `DatabaseSessionService` for Cloud Run agents
- **Connection**: Via VPC Connector (AlloyDB is VPC-native)
- **Schema**: Per-agent namespace within shared instance

### VPC Connector

- **Purpose**: Internal network access from Cloud Run to Memorystore and AlloyDB
- **Region**: `us-central1` (must match all services)

### Cloud Run Configuration

```yaml
# Common settings
min-instances: 0 # Scale to zero in dev
max-instances: 10 # Horizontal scaling limit
cpu: 1 # 1 vCPU per instance
memory: 512Mi # Memory allocation
concurrency: 100 # Max concurrent requests
```

## Security

### Identity-Aware Proxy (IAP)

All public-facing UI endpoints are protected by IAP:

- **Authentication**: Google OAuth 2.0
- **SSO**: IAP cookies scoped to `*.{env}.keynote2026.cloud-demos.goog`
- **WebSocket**: Proxied through BFF (IAP doesn't support WS natively)

### Service-to-Service Authentication

- Cloud Run services use OIDC tokens via `middleware.AttachOIDCToken()`
- Backend agents set to `--no-allow-unauthenticated` on Cloud Run
- Agent Engine agents authenticated via Vertex AI service account

### CORS Configuration

CORS is controlled via the `CORS_ALLOWED_ORIGINS` environment variable:

| Environment | Value                                               |
| :---------- | :-------------------------------------------------- |
| **Local**   | `*` (or unset — defaults to allow all)              |
| **Cloud**   | Explicit comma-separated list of UI service origins |

**Cloud example** (auto-generated by `deploy.py`):

```
https://admin.dev.keynote2026.cloud-demos.goog,https://tester.dev.keynote2026.cloud-demos.goog,https://dash.dev.keynote2026.cloud-demos.goog
```

**Implementation:**

- **Go**: `middleware.CORS()` reads `CORS_ALLOWED_ORIGINS` env var
- **Python**: `serve.py` and `a2a.py` read `CORS_ALLOWED_ORIGINS` env var

**BFF pattern note**: UI services (tester) proxy API and WebSocket calls
through their own origin via a Go BFF. This eliminates most cross-origin issues
in production since all browser requests are same-origin.

## ADK Runtime Services

The `agents/utils/runtime.py` module selects ADK services based on env vars:

| Service      | Local              | Cloud Run                        | Agent Engine            |
| :----------- | :----------------- | :------------------------------- | :---------------------- |
| **Session**  | `InMemorySession`  | `DatabaseSession` (AlloyDB)      | `VertexAiSession`       |
| **Artifact** | `InMemoryArtifact` | `GcsArtifact` (GCS bucket)       | Managed by Agent Engine |
| **Memory**   | `InMemoryMemory`   | None (NPCs are ephemeral)        | `VertexAiMemoryBank`    |

Detection logic:

1. `AGENT_ENGINE_ID` set → Agent Engine services
2. `DATABASE_URL` set → Cloud Run services
3. Neither → Local dev (InMemory)

## Deployment Commands

### Using `deploy.py` (Recommended)

```bash
# Deploy a single service to dev
python scripts/deploy/deploy.py gateway --env dev

# Deploy all services
python scripts/deploy/deploy.py all --env dev

# Build container image + deploy
python scripts/deploy/deploy.py all --build --env dev

# Deploy to production
python scripts/deploy/deploy.py all --project n26-devkey-simulation-prod-a --env prod-a

# Deploy with infrastructure values
# Build image only (no deploy)
python scripts/deploy/deploy.py all --image-only

# Deploy and verify health
python scripts/deploy/deploy.py gateway --verify
```

### Using Cloud Build

```bash
# Deploy to dev
gcloud builds submit \
  --substitutions=_TARGET_PROJECT_ID="n26-devkey-simulation-dev",_ENV_NAME="dev" \
  --config cloudbuild.yaml .

# Deploy to production
gcloud builds submit \
  --substitutions=_TARGET_PROJECT_ID="PROD_PROJECT_ID",_ENV_NAME="prod-a" \
  --config cloudbuild.yaml .
```

## Scaling Considerations

- **Gateway**: Scale based on WebSocket connection count (~1000 per instance)
- **Runner Autopilot**: Scale based on simulation participant count (~100 NPCs per agent)

## Local Development vs Cloud

| Feature  | Local                             | Cloud                     |
| :------- | :-------------------------------- | :------------------------ |
| Pub/Sub  | Emulator (localhost:8103)         | Google Cloud Pub/Sub      |
| Redis    | Docker Container (localhost:8102) | Cloud Memorystore         |
| Database | None (InMemory sessions)          | AlloyDB (DatabaseSession) |
| Auth     | Open (dev-friendly)               | IAP + OIDC                |
| Routing  | Port-based (direct)               | Per-service domains       |
| CORS     | `*` (allow all)                   | Explicit origin allowlist |
| Logging  | Standard Out                      | Cloud Logging             |
